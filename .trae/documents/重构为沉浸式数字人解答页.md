# 重构：沉浸式数字人解答中心

将页面重构为以数字人为核心的解答体验，实现“首屏沉浸解答”与“滚动阅读模式”的无缝切换。

## 1. 布局重构 (Structure)
- **Hero 区域 (新)**：
    - 在页面顶部创建 `hero-section`，高度占屏约 45%。
    - 放置 **超大号赤红之心 (Big Core)**，尺寸放大至 180px+。
    - 背景增加深邃的星云光效，衬托核心。
- **信息流调整**：
    - 标题和元信息移至 Hero 区域下方。
    - **全息控制台**：在非悬浮状态下，作为页面组件直接展示在标题下方，宽度铺满，视觉更舒展。
- **悬浮窗 (Mini Player)**：
    - 保留原有的 `movable-area` 结构。
    - 增加 `wx:if` 逻辑：仅在 `showMiniPlayer` 为 true 时渲染。

## 2. 交互逻辑 (Interaction)
- **滚动监听**：
    - 在 `onPageScroll` (或 `scroll-view` 的 `bindscroll`) 中监听滚动距离。
    - **阈值判断**：
        - 滚动距离 < 300px：**沉浸模式**。显示 Hero 区域的大核心和嵌入式控制台；隐藏悬浮窗。
        - 滚动距离 > 300px：**阅读模式**。Hero 区域淡出/隐藏；右下角浮现悬浮窗 (Mini Player)，接管播放控制。
- **播放状态同步**：确保大核心和小悬浮窗共享同一个 `isPlaying` 和 `audioContext` 状态。

## 3. 样式升级 (Style)
- **Hero 样式**：定义 `.hero-core` 相关样式，增强光晕和粒子密度。
- **嵌入式控制台**：调整 `.holo-console-card` 的样式，使其在嵌入页面时自适应宽度，取消绝对定位或悬浮阴影的某些特性，使其更像页面的一部分。

## 4. 技术路径
1.  修改 `qaDetail.wxml`: 构建 Hero 结构，调整控制台位置。
2.  修改 `qaDetail.wxss`: 适配大尺寸核心和嵌入式控制台，添加过渡动画类。
3.  修改 `qaDetail.js`: 添加滚动监听逻辑，管理 `showMiniPlayer` 状态。
