## 学生管理改造与功能扩展方案

### 一、改造目标

将原有的积分管理功能改造为学生管理功能，并添加以下三个新功能：
1. **账号状态管理**：启用/停用学生账号
2. **部门分配管理**：调整学生所属部门
3. **登录信息管理**：重置学生的登录信息

### 二、整体架构设计

#### 1. 控制器改造

- **新建** `HxStudentController` 替代 `HxPointController`
- **保留**原有积分管理功能，扩展为学生管理
- **路径**：`/hx/student`
- **权限标识**：将 `hx:points:` 改为 `hx:student:`

#### 2. 功能模块设计

| 功能模块 | 现有功能 | 新增功能 |
| ---- | ---- | ---- |
| 学生列表 | 积分账户列表 | 学生完整信息列表 |
| 学生详情 | 积分账户详情 | 学生完整信息详情 |
| 积分管理 | 积分调整 | 保留，增加批量调整 |
| 账号管理 | - | 账号状态管理、部门分配、登录信息重置 |
| 微信管理 | - | 微信绑定状态查看、解绑微信 |
| 密码管理 | - | 重置密码 |

### 三、详细实现方案

#### 1. 控制器实现

**类名**：`HxStudentController`
**路径**：`/hx/student`
**核心依赖**：
- `IHxPointService`：保留积分管理功能
- `ISysUserService`：实现用户管理功能
- `IHxUserWechatService`：实现微信绑定管理

**主要方法**：

| 方法 | URL | 功能 | 权限 |
| ---- | ---- | ---- | ---- |
| `studentList` | `GET /list` | 学生列表查询 | `hx:student:list` |
| `studentInfo` | `GET /{userId}` | 学生详情查询 | `hx:student:query` |
| `updateStudent` | `PUT /` | 更新学生信息 | `hx:student:edit` |
| `resetPwd` | `PUT /resetPwd` | 重置学生密码 | `hx:student:resetPwd` |
| `changeStatus` | `PUT /changeStatus` | 更改账号状态 | `hx:student:changeStatus` |
| `unbindWechat` | `PUT /unbindWechat/{userId}` | 解绑微信 | `hx:student:wechat` |
| `adjustPoints` | `POST /adjustPoints` | 调整积分 | `hx:student:adjustPoints` |
| `resetLoginInfo` | `PUT /resetLoginInfo/{userId}` | 重置登录信息 | `hx:student:edit` |

#### 2. 账号状态管理实现

**功能**：启用/停用学生账号
**实现**：调用 `ISysUserService.changeStatus(SysUser user)`
**参数**：
- `userId`：学生ID
- `status`：账号状态（0正常 1停用）
**权限**：`hx:student:changeStatus`

#### 3. 部门分配管理实现

**功能**：调整学生所属部门
**实现**：直接更新 `SysUser.deptId` 字段
**参数**：
- `userId`：学生ID
- `deptId`：部门ID
**权限**：`hx:student:edit`

#### 4. 登录信息管理实现

**功能**：重置学生的登录信息
**实现**：直接更新 `SysUser` 的登录相关字段
**参数**：
- `userId`：学生ID
**更新字段**：
- `loginIp`：设为 `null` 或 `"-"`
- `loginDate`：设为 `null`
**权限**：`hx:student:edit`

#### 5. 学生信息实体设计

**实体**：`StudentManageVo`（新建）
**包含字段**：
- `SysUser` 核心字段（用户ID、账号、姓名、邮箱、手机号等）
- `HxStudentProfile` 积分字段（当前积分、积分记录等）
- `HxUserWechat` 微信绑定字段（绑定状态、openid等）
- 部门信息（部门名称）
- 角色信息（学生角色标识）

#### 6. 服务层实现

**扩展** `IHxPointService` 或 **新建** `IHxStudentService`，提供以下方法：
- `selectStudentList`：查询学生列表
- `selectStudentInfo`：查询学生详情
- `updateStudent`：更新学生信息
- `changeStudentStatus`：更改学生账号状态
- `resetStudentLoginInfo`：重置学生登录信息

### 五、页面调整建议

1. **列表页面**：
   - 增加学生基本信息字段（账号、姓名、邮箱、手机号等）
   - 增加部门、角色、账号状态等字段
   - 增加操作按钮：编辑、重置密码、调整积分、解绑微信、更改状态

2. **详情页面**：
   - 分标签展示：基本信息、积分信息、微信绑定、登录记录
   - 增加编辑按钮

3. **编辑页面**：
   - 包含基本信息、部门分配、账号状态等字段
   - 不显示头像上传字段

### 六、技术实现要点

1. **学生信息整合**：将 `SysUser`、`HxStudentProfile`、`HxUserWechat` 信息整合到一个VO中
2. **权限控制**：确保所有操作都有适当的权限控制
3. **事务管理**：批量操作和更新操作需要事务支持
4. **日志记录**：所有操作都需要记录详细日志
5. **数据校验**：对所有输入参数进行严格校验
6. **错误处理**：提供友好的错误提示信息

### 七、实现步骤

1. **新建** `StudentManageVo` 实体
2. **新建** `HxStudentController` 控制器
3. **扩展**服务层方法
4. **配置**权限标识
5. **调整**前端页面
6. **测试**所有功能

### 八、预期效果

1. **功能完整**：实现学生的全面管理，包括基本信息、账号状态、部门分配、积分管理、微信绑定等
2. **权限明确**：不同功能对应不同的权限标识，便于权限管理
3. **操作便捷**：提供批量操作功能，提高管理效率
4. **界面友好**：清晰的列表和详情页面，便于查看和管理
5. **符合要求**：学生不设置头像，满足用户需求

通过以上改造和扩展，可以将原有的积分管理功能升级为完整的学生管理功能，满足用户对学生管理的各种需求。
