# 实现文件管理系统及统一文件上传功能（Linux兼容版）

## 1. 需求分析

* 实现单独的文件上传业务功能，处理上传的文件
* 建立文件管理系统，分类管理上传的文件（封面、音频、视频等）
* 统一活动管理、故事管理、问答管理的文件上传功能
* 将问答管理的视频链接输入改为直接上传
* 确保文件目录符合Linux规范，便于从Windows切换到Linux修改路径

## 2. 系统设计

### 2.1 跨平台文件路径处理

* 使用相对路径存储文件，避免硬编码绝对路径
* 利用若依框架的文件上传工具类，确保跨平台兼容
* 配置文件路径支持环境变量，便于在不同环境下切换

### 2.2 数据库设计

#### 文件表（hx_file）

| 字段名 | 类型 | 描述 |
|-------|------|------|
| file_id | bigint | 文件ID |
| file_name | varchar(255) | 存储文件名 |
| original_name | varchar(255) | 原始文件名 |
| file_path | varchar(500) | 相对文件路径（不包含根目录） |
| file_url | varchar(500) | 访问URL |
| file_type | varchar(50) | 文件类型（image/audio/video/document） |
| file_category | varchar(50) | 文件分类（cover/audio/video等） |
| content_type | varchar(50) | 文件MIME类型 |
| file_size | bigint | 文件大小（字节） |
| upload_time | datetime | 上传时间 |
| upload_by | varchar(64) | 上传者 |
| business_type | varchar(50) | 关联业务类型（activity/story/qa） |
| business_id | bigint | 关联业务ID |
| business_field | varchar(50) | 关联业务字段（coverUrl/videoUrl等） |
| status | char(1) | 状态（0：正常，1：删除） |
| remark | varchar(500) | 备注 |

## 3. 后端实现

### 3.1 配置调整

* 在 `application.yml` 中配置文件上传路径，支持环境变量替换
* 确保文件上传工具类使用相对路径，避免平台差异

### 3.2 实体类

* 创建 `HxFile` 实体类，对应文件表
* 使用 `@TableField` 注解处理字段映射

### 3.3 Mapper

* 创建 `HxFileMapper` 接口及XML文件
* 实现文件的增删改查，支持按业务类型、业务ID、文件分类等条件查询

### 3.4 Service

* 创建 `IHxFileService` 接口及实现类
* 实现文件上传、下载、删除等核心功能
* 处理文件与业务的关联关系

### 3.5 Controller

* 创建 `HxFileController`，提供文件管理的REST API
  * 文件上传接口：`/hx/file/upload`
  * 文件列表查询：`/hx/file/list`
  * 文件删除接口：`/hx/file/delete`
  * 文件详情查询：`/hx/file/{fileId}`

## 4. 前端实现

## 5. 部署考虑

### 5.1 Linux环境配置

* 配置nginx静态资源映射，指向文件存储目录
* 确保Java应用有文件读写权限
* 配置文件上传路径为Linux兼容格式

### 5.2 跨平台切换

* 配置文件路径支持环境变量，便于在Windows和Linux之间切换
* 使用相对路径存储文件，避免硬编码绝对路径
* 确保文件路径分隔符使用统一格式
