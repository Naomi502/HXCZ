### 二、创建文件清理工具类

1. **创建文件清理工具类**
   - 文件：`/src/main/java/com/ruoyi/system/utils/FileCleanupUtils.java`
   - 核心功能：
     - 扫描服务器文件目录
     - 从数据库中提取所有引用的文件URL
     - 对比找出未引用的文件
     - 安全删除未引用文件

2. **核心方法设计**
   - `scanDirectory(String directoryPath)`：递归扫描指定目录下的所有文件
   - `extractAllReferencedUrls()`：从所有表中提取文件引用URL
   - `getUnusedFiles()`：获取未引用的文件列表
   - `cleanupUnusedFiles()`：执行清理操作
   - `extractUrlsFromHtml(String html)`：从HTML内容中提取文件URL

### 三、实现引用提取逻辑

1. **涉及表和字段**
   - `HxActivity`：`coverUrl`、`description`
   - `HxActivityParticipant`：`proofImageUrl`
   - `HxStory`：`coverUrl`、`content`
   - `HxQa`：`videoUrl`、`content`
   - `HxTreeHole`：`content`

2. **HTML内容处理**
   - 使用正则表达式提取 `<img>`、`<video>`、`<audio>` 等标签中的URL
   - 支持相对路径和绝对路径
   - 处理不同格式的URL（完整URL、服务器相对路径）

### 四、创建定期清理任务

1. **创建定时任务类**
   - 文件：`/src/main/java/com/ruoyi/system/task/FileCleanupTask.java`
   - 使用 `@Scheduled` 注解实现自动定期清理
   - 默认每天凌晨2点执行

2. **配置项**
   - 文件保留期（默认7天）
   - 白名单目录（默认包含静态资源目录）
   - 清理开关
   - 日志级别

### 五、安全清理机制

1. **文件保留期**：默认保留最近7天的文件，避免误删刚上传的文件
2. **白名单机制**：配置重要目录或文件，避免误删
3. **详细日志**：记录清理时间、扫描文件数、删除文件数、删除文件列表
4. **渐进式清理**：首次运行只生成报告，不实际删除；后续可配置自动清理
5. **事务支持**：确保数据库操作的完整性

### 六、技术实现细节

1. **文件扫描**：使用Java NIO进行高效的文件扫描
2. **正则表达式**：使用Pattern和Matcher类提取HTML中的URL
3. **日期处理**：使用LocalDateTime处理文件创建时间
4. **配置管理**：使用Spring的@Value注解读取配置
5. **日志记录**：使用SLF4J记录详细日志

### 七、预期效果

1. 管理页面移除封面/视频时，只清空表单字段，不立即删除文件
2. 系统每天自动扫描所有文件
3. 分析所有表中的文件引用关系
4. 删除未被任何内容引用且超过保留期的文件
5. 生成详细的清理日志，便于追溯
6. 支持灵活配置，适应不同场景需求

### 八、注意事项

1. 确保清理任务不会影响系统正常运行
2. 保留足够的文件保留期，避免误删重要文件
3. 定期检查清理日志，确保清理功能正常工作
4. 对于大型系统，考虑分批次清理，避免长时间占用系统资源
5. 添加适当的异常处理，确保任务执行稳定性
