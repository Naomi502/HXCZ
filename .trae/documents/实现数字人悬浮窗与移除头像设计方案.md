# 实现“数字人”悬浮窗与移除 AI 头像方案

## 1. 移除 AI 气泡头像
- **修改 `qa.wxml`**：在消息列表循环中，当角色为 AI (`role !== 'user'`) 时，**删除**原有的 `<image class="avatar" ...>` 代码。
- **修改 `qa.wxss`**：调整 `.message.left .content` 的样式，移除因头像存在而设置的左侧间距，使气泡自然靠左排列。

## 2. 添加悬浮“数字人”组件
- **修改 `qa.wxml`**：
  - 在页面顶部（`header-bar` 下方或同层级）添加一个悬浮容器 `digital-human-container`。
  - 在容器内放置一个 `<video>` 组件。
  - **无缝切换逻辑**：为了避免视频切换时的黑屏或加载卡顿，我们将使用 **“GIF 待机 + 视频说话”** 或 **“单视频源切换”** 的策略。考虑到小程序性能，推荐使用 **单 Video 组件切换 `src`** 的方式，并配合 `poster`（封面图）过渡。
  - **去黑底**：给 video 组件应用 CSS 滤镜 `mix-blend-mode: screen`（滤色模式），这能让黑色背景变透明，仅保留玩偶主体（适合浅色/彩色背景页面）。

## 3. 逻辑控制 (`qa.js`)
- **新增数据字段**：
  - `idleVideoUrl`: 待机视频链接（请您后续填入）。
  - `speakingVideoUrl`: 说话视频链接（请您后续填入）。
  - `currentVideoUrl`: 当前播放的视频。
- **状态监听**：
  - 在 `isSpeaking` 变为 `true` 时，将 `currentVideoUrl` 设为 `speakingVideoUrl` 并自动播放。
  - 在 `isSpeaking` 变为 `false` 时，将 `currentVideoUrl` 设为 `idleVideoUrl` 并循环播放。

## 4. 样式调整 (`qa.wxss`)
- 设置悬浮窗固定在屏幕上方中间或右上角（根据您的设计偏好，默认置于 Header 下方居中）。
- 添加 `mix-blend-mode: screen` 样式以去除黑色背景。
