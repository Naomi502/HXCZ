# 项目改进计划

## 一、需求分析

用户要求可以监控每个用户的实际操作，并且管理员和其他用户分开展示操作日志。基于若依框架现有的操作日志功能，我们需要进行以下优化：

1. 扩展操作日志功能，增加用户角色区分
2. 创建新的操作日志视图，区分管理员和普通用户
3. 增强操作日志的查询和筛选功能
4. 增加操作日志的统计分析功能
5. 修复安全隐患

## 二、现有功能分析

### 1. 操作日志功能

若依框架已经实现了基本的操作日志功能：
- 使用AOP记录操作日志
- 支持查询、删除、清空操作日志
- 记录了操作模块、业务类型、请求方法、操作人、操作时间等信息

但是，现有功能缺少：
- 用户角色区分
- 管理员和普通用户分开的操作日志视图
- 灵活的查询条件
- 统计分析功能

### 2. 安全隐患

- SSL验证被禁用（TrustAnyTrustManager和TrustAnyHostnameVerifier）
- token密钥硬编码在application.yml中

## 三、改进方案

### 1. 操作日志功能优化

#### 1.1 扩展操作日志实体类

在`SysOperLog`类中增加用户ID和角色ID字段，便于区分不同用户和角色的操作：

```java
/** 用户ID */
private Long userId;

/** 角色ID */
private Long roleId;
```

#### 1.2 优化操作日志记录逻辑

在`LogAspect`类中，记录操作日志时，同时记录用户ID和角色ID：

```java
if (loginUser != null) {
    operLog.setOperName(loginUser.getUsername());
    operLog.setUserId(loginUser.getUser().getUserId());
    if (loginUser.getUser().getRoles() != null && !loginUser.getUser().getRoles().isEmpty()) {
        operLog.setRoleId(loginUser.getUser().getRoles().get(0).getRoleId());
    }
    SysUser currentUser = loginUser.getUser();
    if (StringUtils.isNotNull(currentUser) && StringUtils.isNotNull(currentUser.getDept())) {
        operLog.setDeptName(currentUser.getDept().getDeptName());
    }
}
```

#### 1.3 优化操作日志查询功能

在`SysOperlogController`中增加按用户ID和角色ID查询的功能，并实现普通用户只能查看自己操作日志的权限控制：

```java
@PreAuthorize("@ss.hasPermi('monitor:operlog:list')")
@GetMapping("/list")
public TableDataInfo list(SysOperLog operLog) {
    // 普通用户只能查看自己的操作日志
    LoginUser loginUser = SecurityUtils.getLoginUser();
    if (!loginUser.getUser().isAdmin()) {
        operLog.setUserId(loginUser.getUser().getUserId());
    }
    startPage();
    List<SysOperLog> list = operLogService.selectOperLogList(operLog);
    return getDataTable(list);
}
```

#### 1.4 增加操作日志统计分析功能

在`ISysOperLogService`接口中增加统计分析方法，并在实现类中实现：

```java
/**
 * 统计操作日志
 * 
 * @param operLog 操作日志对象
 * @return 统计结果
 */
public Map<String, Object> statOperLog(SysOperLog operLog);
```

#### 1.5 创建新的操作日志视图

在`ruoyi-ui/src/views/monitor/operlog/`目录下创建以下文件：

- `admin-index.vue`：管理员操作日志视图，展示所有用户的操作日志
- `user-index.vue`：普通用户操作日志视图，只展示当前用户的操作日志

### 2. 安全隐患修复

#### 2.1 修复SSL验证问题

替换`TrustAnyTrustManager`和`TrustAnyHostnameVerifier`，实现正确的SSL验证：

```java
// 删除TrustAnyTrustManager和TrustAnyHostnameVerifier类
// 使用默认的SSL验证机制
```

#### 2.2 优化token密钥配置

将token密钥从硬编码改为配置文件或环境变量：

```yaml
# application.yml

  # 令牌密钥，生产环境请使用随机生成的密钥
  secret: ${TOKEN_SECRET:abcdefghijklmnopqrstuvwxyz}
```
